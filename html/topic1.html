Understanding HTTP

Early Development

HTTP was developed in the early 1990s as a simple way to share documents over the Internet. HTTP/0.9 supported only basic GET requests and plain HTML responses. Later, HTTP/1.0 introduced headers and metadata, improving communication between client and server.


HTTP Evolution

HTTP/0.9 (1990)

Extremely simple, GET-only protocol. No headers or status codes. Returned raw HTML, requiring a new TCP connection per request.

HTTP/1.0 (1996)

Introduced headers, metadata, and MIME types. Still used one TCP connection per request.

HTTP/1.1 (1999)

Persistent connections, cache control, host headers, pipelining, and more efficient request handling.



HTTP/0.9

HTTP/0.9, developed in 1990, was extremely simple. It supported only the GET method and returned raw HTML, without headers or status codes. Each request required a new TCP connection, and there was no way to describe content type, caching rules, or other metadata. This simplicity reflected the early web, designed mainly for sharing static documents.

HTTP/1.0

Introduced in 1996, HTTP/1.0 added headers and metadata, enabling servers to describe content type, caching rules, and other information. Despite this improvement, a new TCP connection was still required for each request. HTTP/1.0 allowed more flexible content delivery and laid the groundwork for status codes and MIME types.

HTTP/1.1

Released in 1999, HTTP/1.1 introduced persistent connections, allowing multiple requests over a single TCP connection. It enhanced cache control, introduced flexible methods, and improved support for proxies and gateways. Host headers enabled virtual hosting, and error handling became more reliable. HTTP/1.1 also enabled pipelining, which reduced latency for multiple requests in sequence.
