Understanding HTTP

Early Development

HTTP was developed in the early 1990s as a simple way to share documents over the Internet. HTTP/0.9 supported only basic GET requests and plain HTML responses. Later, HTTP/1.0 introduced headers and metadata, improving communication between client and server.

HTTP Evolution

HTTP was developed to share data on the web and enable resource access across the Internet. The earliest version, HTTP/0.9 (1990), supported only the GET method and sent raw HTML, without headers or status codes, showing the simplicity of the early web (Fielding et al., 1999).

HTTP/1.0 (RFC 1945) added MIME-like headers and metadata, enabling servers to describe content type, caching rules, and other information. However, it still opened a new TCP connection per request and had weak caching and virtual hosting support (Berners-Lee, Fielding, & Nielsen, 1996).

HTTP/1.1 (RFC 2616) improved efficiency by allowing multiple requests over a single TCP connection, enhanced cache control, flexible methods, and better support for proxies and gateways. Host headers allowed virtual hosting, and error handling became more reliable (Fielding et al., 1999).

Overall, the progression from HTTP/0.9 → 1.0 → 1.1 shows step-by-step improvements based on real-world needs. Updates between 1996-1999 clarified syntax, headers, and ensured backward compatibility (W3C, 1999), turning HTTP into a flexible protocol capable of supporting the growing complexity of the World Wide Web.

Version Year Key Features
HTTP/0.9 1990 Simple GET requests
HTTP/1.0 1996 Headers, Metadata, MIME types
HTTP/1.1 1999 Persistent connections, caching, host headers

Key Points

  • HTTP/0.9 - simple, GET only, no headers or status codes
  • HTTP/1.0 - added headers and metadata, still one TCP per request
  • HTTP/1.1 - persistent connections, cache control, host headers, improved efficiency
  • Evolution driven by real-world web needs and collaborative standardization
